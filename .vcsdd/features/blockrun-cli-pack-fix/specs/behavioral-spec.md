# blockrun-cli-pack-fix — Behavioral Specification (EARS)

Feature: `blockrun-cli-pack-fix` · Mode: lean · Language: TypeScript

**HOTFIX, priority over everything else in flight.** Every published `blockrun-cli` version (1.1.0,
1.2.0) crashes at startup for EVERY user who installs via `npm install -g blockrun-cli` or
`npx blockrun-cli` — confirmed live, reproduced independently in this session:

```
$ npm pack --pack-destination /tmp/pack-verify && cd /tmp/pack-verify && tar xzf blockrun-cli-1.2.0.tgz \
  && cd package && node dist/index.js --version
Error: cost-model: could not locate src/commands/ from /private/tmp/pack-verify/package/dist
    at findCommandsDir (file:///.../dist/index.js:3375:9)
```

Root cause: `src/core/cost-model.ts`'s `COMMAND_COST_MODEL` is computed via a TOP-LEVEL
`deriveCostModel()` call that reads `src/commands/*.ts` from disk AT MODULE LOAD TIME (`readFileSync`/
`readdirSync`, resolved relative to the running module's own location). `package.json`'s `"files":
["dist", "README.md"]` means an npm-installed package NEVER ships `src/` at all — so
`findCommandsDir()` always throws, crashing the ENTIRE CLI (every command, including `--version`)
before a single line of user code runs. Previous manual verification (`npm install -g .`) was a FALSE
POSITIVE: that command creates a symlink into the repo checkout itself (where `src/` genuinely IS a
sibling of `dist/`), masking the exact failure a REAL npm-registry install exhibits.

Every requirement below is grounded in this LIVE, independently-reproduced crash — not assumed.

---

## 0. Non-goals

- REQ-PACK-NG-001: THIS FEATURE SHALL NOT change `COMMAND_COST_MODEL`'s VALUES for any of the 18
  commands (`wallet`/`models`/`dex` stay `"free"`, the other 15 stay `"paid"`) — this is a packaging
  fix, not a behavior change.
- REQ-PACK-NG-002: THIS FEATURE SHALL NOT change `blockrun commands --json`'s output shape or any other
  command's behavior.

## 1. Build-time cost-model generation (replaces runtime `src/` filesystem access)

- REQ-PACK-001: `COMMAND_COST_MODEL`'s derivation (which commands call `gatePaidCall(`) SHALL move from
  a RUNTIME `readFileSync`/`readdirSync` scan (which requires `src/commands/*.ts` to exist on disk next
  to the running code — true only in this repo's own checkout, NEVER true for an npm-installed package)
  to a BUILD-TIME generation step: a new `scripts/generate-cost-model.mjs` greps the REAL
  `src/commands/*.ts` files (the SAME `gatePaidCall(` ground truth REQ-DX-001/003 already established —
  values UNCHANGED, REQ-PACK-NG-001) and writes a plain, dependency-free TypeScript module,
  `src/core/cost-model.generated.ts`, exporting `COMMAND_COST_MODEL` as a literal object — NO
  `readFileSync`/`readdirSync`/`import.meta.url` path resolution anywhere in the generated file or
  anything it's bundled with, so the built `dist/index.js` never needs `src/` to exist at runtime again.
- REQ-PACK-002: `src/core/cost-model.ts` (the module `commands-catalog.ts` already imports) SHALL become
  a thin re-export of `src/core/cost-model.generated.ts` — its PUBLIC API (`COMMAND_COST_MODEL`,
  `CostModel` type) SHALL NOT change, so no other file needs to change its own imports.
- REQ-PACK-003: `package.json` SHALL gain a `"prebuild"` script (`node scripts/generate-cost-model.mjs`)
  — npm's own standard pre-script convention, auto-invoked by `npm run build` with no extra step —
  guaranteeing `cost-model.generated.ts` is ALWAYS freshly regenerated before `tsup` bundles `dist/
  index.js`, so a published build can never ship a stale mapping.
- REQ-PACK-004: `src/core/cost-model.generated.ts` SHALL be COMMITTED to git, not gitignored — DECISION,
  with reasoning recorded here per Dais/team-lead's explicit request: this repo's `npm test` runs `tsx
  --test` directly against TypeScript SOURCE, with NO prior build step — a gitignored generated file
  would be ABSENT on a fresh clone until someone thinks to run the generator manually, breaking `npm
  test` for that case. Committing it also makes any accidental drift (someone edits a command's
  `gatePaidCall` usage without regenerating) VISIBLE as a git diff at review time, which a gitignored
  file cannot provide. The file's own header comment SHALL say "AUTO-GENERATED by
  scripts/generate-cost-model.mjs — do not hand-edit."
- REQ-PACK-005: A NEW Tier-1 test SHALL assert NO DRIFT: independently re-derive the `gatePaidCall(`
  -grep mapping fresh from the REAL `src/commands/*.ts` files (in-test, the SAME anti-drift pattern
  `blockrun-cli-agent-dx`'s `introspect-schema.test.ts` already established — an independent reference
  computation, not a call into the code under test) and assert it is IDENTICAL to the COMMITTED
  `cost-model.generated.ts`'s `COMMAND_COST_MODEL` — catching a forgotten regeneration BEFORE it ships,
  not just at `npm run build` time.

## 2. Packaged-artifact regression test (the test that would have caught this bug)

- REQ-PACK-006: A NEW Tier-2 test SHALL run `npm pack` to produce a REAL tarball, extract it to a fresh
  temp directory (mirroring EXACTLY how `npm install`/`npx` would lay out the installed package — NOT
  `npm install -g .`'s symlink-into-the-repo shortcut, which is what produced the false-positive
  verification that let this bug ship), and execute `node <extracted>/dist/index.js --version` AND
  `node <extracted>/dist/index.js commands --json` from WITHIN that isolated directory (so no sibling
  `src/` can accidentally be found by a relative-path fallback) — both SHALL exit `0`. This is the
  regression test that would have caught the ORIGINAL bug and must never regress again.

## 3. Cross-cutting / regression

- REQ-PACK-007: THE existing test suite (`npm test`, 553 tests as of this feature's start) SHALL remain
  100% green throughout and at the end of this feature.
- REQ-PACK-008: `execution-notes.md` SHALL record, honestly, the false-positive verification method
  (`npm install -g .`'s symlink) that let this bug reach 2 published versions undetected — not just the
  fix, so the SAME mistake (verifying via a local symlink install instead of a real packed tarball) is
  not repeated by a future feature.
